# /refactor-safe

## 手順

1. 対象($ARGUMENTS)の現在の振る舞いを把握し、テストが無ければ先に特性テストを足す。
2. テストがグリーンであることを確認する(リファクタ前の基準点)。
3. 小さなステップでリファクタする(命名・重複除去・分割)。各ステップ後にテストを実行。
4. 公開インターフェース(関数シグネチャ・戻り値の型)は原則維持する。
5. 完了後、全関連テストがグリーンであることを確認する。

## 出力フォーマット

- 変更の意図(なぜ)・適用したリファクタの種類。
- リファクタ前後でテストが通っている証跡。

## ルール

- 振る舞いを変えない(機能追加・バグ修正は別 PR)。
- 要求外の過剰な抽象化をしない(`.claude/rules/`)。
- `export default` 化や `any` 導入で「簡単に」しない。
