# /dev

## 手順

1. `pnpm dev`(Turbopack)を起動する。既に起動中なら再利用する。
2. 起動ログを確認し、コンパイルエラー/警告が無いか見る。
3. 引数のルート($ARGUMENTS、未指定なら `/`)をブラウザで開いて表示を確認する。
4. ハイドレーションエラーが出たら原因を切り分ける(`HYDRATION_ERROR_FIX.md` の観点)。

## 出力フォーマット

- 起動 URL、確認したルート、観察結果(正常/エラー)を簡潔に。
- エラーがあれば原因の仮説と次の一手。

## ルール

- UI 変更は型/テストだけでなく実画面で確認する。確認できない場合は「未確認」と明記。
- 固定 `sleep` で待たず、起動完了をログで判断する。
