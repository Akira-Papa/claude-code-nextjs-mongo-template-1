# サバキャン日本語スタイル

## いつ使うか

- このテンプレの教材・README・SKILL.md・解説文を書く/直すとき。
- 日本語ドキュメントのトーンと用語を統一したいとき。

## やること

- 「なぜ・何・どう」を必ず含める(理由 → 内容 → 手順/例)。
- 読者は「Next.js は触れるが Claude Code 運用は初めて」の開発者と想定。
- 1 文を短く。専門語は初出で 1 行補足する。
- コード/識別子は英語、説明は日本語。コード片は最小で動く形に。
- 断定する前に根拠(計測・公式仕様)を添える。捏造しない。

## 守るルール

- ✅ 見出しは H1 から開始し、構造を一貫させる。
- ✅ ❌/✅ で「やってはいけない/推奨」を対比して示す。
- ✅ 長くしすぎない(README は要点 100–200 行、詳細は別ファイルへ)。
- ❌ 「いい感じに」など曖昧表現で逃げない。
- ❌ 絵文字の乱用・過剰な装飾をしない(✅❌ と最小限の記号に留める)。

## 典型例(語り口)

```text
悪い: 「これを設定するといい感じに動きます。」
良い: 「`server-only` を付けると、誤ってクライアントに import した時点で
      ビルドが失敗する。秘密の漏洩を“実行前”に止められるからこうする。」
```

## アンチパターン

- 結論を先に言わず、長い前置きで読者を待たせる。
- 英語をそのまま訳さず直訳調で読みにくくする。
- 確認していない事実を断定的に書く(必ず根拠 or 「未確認」と明示)。
