# セキュリティルール

## シークレット

- ❌ 秘密(API キー・接続文字列・署名鍵)を `.env.example` 以外に書かない・コミットしない。
- ✅ `.env.local`(git 管理外)に実値。`.env.example` はプレースホルダのみ。
- ✅ 秘密を扱うモジュールは `import 'server-only'`。公開可の値だけ `NEXT_PUBLIC_`。
- ❌ 秘密を `console.log` / レスポンス / エラーメッセージに含めない。

## 入力検証

- ✅ ランタイム境界(API 入力・env・外部/DB 応答)は必ず Zod で検証。
- ✅ ユーザー入力は型強制してからクエリ・コマンドに渡す(NoSQL Injection 防御)。
- ❌ 検証前の生データを DB / シェル / レンダリングへ流さない。

## 認証 / 認可

- ✅ 認証(誰か)と認可(何をしてよいか)を分けて両方チェック。
- ✅ 保護は server 側(ページ + Server Action / Route Handler)で行う。クライアント表示制御だけに頼らない。
- ✅ セッション/トークンは httpOnly Cookie。

## エラー出力

- ✅ ユーザー向け要約と内部ログを分離。内部構造(スタック・DB エラー文)を露出しない。

## 依存 / 外部スキル

- ✅ 外部スキルは取り込み前に antivirus スキャン(`install.sh` の構成)。
- ✅ 取り込んだ依存・スキルのライセンスと出典を記録する。
