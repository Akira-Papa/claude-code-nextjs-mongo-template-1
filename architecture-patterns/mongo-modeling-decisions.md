# MongoDB モデリング判断の記録

スキーマ・インデックス設計の判断をここに残す。新しいコレクションを追加したら追記する。

## 設計の原則(このプロジェクトの既定)

- アクセスパターン駆動。「どう読むか/書くか」からスキーマを決める。
- 埋め込み: 一緒に読む・1 対少数・更新が一体。参照: 独立して伸びる・多対多。
- 複合インデックスは ESR(Equality → Sort → Range)順。
- 読み取り専用は `.lean()`。新規/変更クエリは `.explain()` で IXSCAN 確認。

## コレクション別の記録(例フォーマット)

### `users`

- 主なクエリ: `email` で検索 / `role` + `createdAt` で一覧。
- 判断: `email` に unique index。一覧用に `{ role: 1, createdAt: -1 }`(ESR)。
- 埋め込み/参照: プロフィール詳細は埋め込み、投稿は参照(独立して増えるため)。
- explain: 一覧クエリ IXSCAN 確認済み(examined ≒ returned)。
