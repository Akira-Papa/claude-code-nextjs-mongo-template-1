# セキュリティレビューの依頼

```text
# セキュリティレビュー: <対象範囲>

## 対象
- <ファイル/ディレクトリ or "現在の差分">
- 特に気になる点: <認証/入力/外部連携 など、あれば>

## 観点(security-auditor に委譲してよい)
1. 入力検証: API 入力・env・DB 応答が Zod を通っているか
2. NoSQL Injection: find({...req.body}) / $where / 入力でフィールド名決定
3. 認証 / 認可: 認可(所有者・ロール)の漏れ、server 側で守っているか
4. シークレット: NEXT_PUBLIC_ 誤用、ログ/レスポンスへの漏洩、server-only 欠落
5. API ルート: レート制限・サイズ上限・エラー漏洩

## 出力
- 重大度(Critical/High/Medium/Low)付きの所見 + file:line + 修正案。
- 確証がないものは「要確認」と明記。発見と修正は分けて。
```
