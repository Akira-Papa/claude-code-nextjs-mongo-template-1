# 新機能の実装依頼

以下をコピーして `<...>` を埋めて使う。

```text
# 機能: <機能名>

## 背景 / 目的
<なぜこの機能が必要か。誰のどんな課題を解くか>

## 要件
- <満たすべき振る舞い 1>
- <満たすべき振る舞い 2>
- 受け入れ条件: <これが満たされたら完成>

## 制約
- スタック: Next.js 16 / Mongoose / Zod(CLAUDE.md 準拠)
- <その他の制約。既存 API・データ構造など>

## 進め方
1. まず計画を提示して(変更ファイル・データフロー・テスト方針)。実装はその後。
2. Server Component デフォルト、入力境界は Zod、認可チェックを忘れず。
3. 関連テスト(ユニット/必要なら E2E)を追加し、グリーンを確認。
4. 完了後、変更点と検証方法を簡潔に報告(未確認は「未確認」と明記)。
```
