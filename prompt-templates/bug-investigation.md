# バグ調査の依頼

```text
# バグ: <症状を一言で>

## 再現手順
1. <操作 1>
2. <操作 2>
→ 期待: <期待する挙動> / 実際: <実際の挙動>

## 環境 / 情報
- 発生箇所(分かれば): <URL/ルート/ファイル>
- エラーメッセージ / ログ: <あれば貼る>
- 発生頻度: <常に / たまに / 特定条件>

## 進め方
1. まず最小再現を作って原因を特定して(対症療法でなく根本原因)。
2. 仮説 → 検証の順で、証跡(ログ/explain/テスト)を示しながら進めて。
3. 修正は最小限に。関連テストを追加して再発を防いで。
4. 原因と修正内容を「なぜそうなっていたか」を含めて報告して。
```
