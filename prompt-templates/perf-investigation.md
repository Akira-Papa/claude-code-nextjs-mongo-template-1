# パフォーマンス調査の依頼

```text
# パフォーマンス: <遅い箇所>

## 症状
- <どの操作が遅いか。体感 or 計測値>
- 目標: <許容したい応答時間 / 指標>

## 情報
- 対象ルート/クエリ: <URL or コード片>
- データ規模: <件数の目安>

## 進め方
1. まず計測して(推測で最適化しない)。DB なら .explain('executionStats')、
   フロントなら Core Web Vitals / バンドルサイズ。
2. ボトルネックを 1 つ特定し、影響の大きい順に対処して。
3. MongoDB は ESR 複合インデックスと IXSCAN 化を優先。N+1 や COLLSCAN を潰して。
4. 改善は before/after の計測値で示して(「速くなるはず」は不可)。
```
